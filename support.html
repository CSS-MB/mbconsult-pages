What to include

  • Tenant identifiers (NetSuite Account ID; Entra tenant ID if applicable)
  • Feature area (Outlook add-in, SharePoint sync, Copilot agent, Connector)
  • Timestamp and correlation/trace IDs (Graph/NetSuite/Function logs)
  • Exact error text and steps to reproduce

Self-service checks

  1. Verify the integration role in NetSuite has required permissions and token/OAuth access.
  2. Confirm Entra app registrations, API permissions, and consent are complete.
  3. If using Azure Functions/Storage, check function logs and app settings.
  4. Re-run connectivity tests; ensure firewall/IP allowlists aren't blocking outbound calls.

Common Issues & Solutions

Authentication Problems

  • OAuth token expired: Re-authenticate through the integration setup
  • Permission denied: Verify admin consent has been granted for required Microsoft Graph permissions
  • NetSuite access issues: Check that the integration role has the necessary permissions

Sync Issues

  • Data not syncing: Check Azure Function logs for error messages
  • Partial sync: Verify field mappings and data type compatibility
  • Sync delays: Review trigger configurations and scheduling settings

Configuration Problems

  • Missing data fields: Update field mappings in the integration configuration
  • Incorrect formatting: Review data transformation rules
  • Connection timeouts: Check network connectivity and firewall settings
